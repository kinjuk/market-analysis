# Valuation Methodology & Philosophy

This directory marks a shift in how valuation data is processed and presented, moving from legacy Excel models to a streamlined HTML-based environment.

## 🟢 The Transition: From Excel to HTML/JS

While Microsoft Excel remains the gold standard for financial logic, this project adopts an **integrated HTML and Script-based approach** to improve the delivery of our analysis. 

### Why the Change?
Excel is a powerful tool, but this shift addresses limitations in modern data presentation:

*   **Efficiency:** Automated scripts handle complex formulas, reducing manual overhead and minimizing calculation errors.
*   **Readability:** By decoupling logic from the presentation layer, the final analysis is clean and professional, removing the visual clutter of a spreadsheet grid.
*   **Data Presentation:** The web-native environment allows for dynamic layouts and polished visualizations that provide a superior "show and tell" experience.

## 🛠 Design Intent: Narrated Analysis

The primary goal of this environment is to provide a **structured narrative of the valuation**. 

Unlike a spreadsheet, which often functions as an open-ended "scratchpad," this HTML-based approach is intentionally optimized for **clarity and insight delivery**. While the underlying scripts implement rigorous financial formulas (DCF, WACC, Multiples), the interface is designed to focus on the results and the "story" behind the numbers.

*Note: While a technical user can inspect the implementation logic within the scripts, the environment is prioritized for high-fidelity presentation rather than raw interactivity.*
